InterentOfThingsProject:
启动程序--->欢迎界面--->加载网络--->成功进入主界面，失败弹出对话框；
点击重试，随机生成数字,t通过Handler处理，为1，进入主界面，为2,，继续弹出对话框...
点击退出，退出程序；

主界面--->点击ListView的Item--->弹出设备控制对话框--->显示当前设备的一些具体信息；
点击关机，对话框关闭，当前Item为灰色；
点击取消，对话框关闭；

主界面--->长按ListView的Item--->弹出设备编辑的对话框--->编辑当前设备的一些信息，唯独设备ID不可编辑；
点击保存，保存当前信息，存入数据库，以便下次重启程序时，从数据库内获取数据；
点击取消，关闭对话框，不保存数据，跟之前一样；

主界面--->点击删除按钮，刷新ListView，显示CheckBox选中按钮，点击ListView的Item或者点击CheckBox选择设备，再一次点击删除按钮，即可删除当前选中的设备数据；

主界面--->点击添加按钮
--->弹出添加设备的对话框，输入相关设备信息；
点击保存，生成一个设备对象，保存这个对象到数据库中，并刷新ListView；
点击取消，关闭对话框；

主界面--->点击空中配网按钮--->弹出Toast消息，未做处理；

主界面--->点击菜单按钮--->弹出对话框，第一次弹出对话框，配置数据是在res/raw中的.Properties配置文件加载的，其他则从SharePreference中加载的；（在res/raw中的.Properties配置文件，Android是通过R.raw.config来获取输入流的，但是无法获取输出流）
添加相关数据之后，点击保存，会把数据存入SharePreference中，以便下次重新获取数据；
点击取消，关闭对话框，不做保存操作；

ListView底部的动态进度条，采用Handle和Timer，每秒发送一次数据，当数据大于100是，清零处理，实际处理和采用Thread和Handle做处理，或者采用Service；
